# cardscentral.github.io

This is the **GitHub Pages site repository** for the Cards Central organization.
It is served at the org root:

- **Landing** → https://cardscentral.github.io/ (marketing page)
- **Prod PWA** → https://cardscentral.github.io/app/ (latest published release)
- **QA PWA** → https://cardscentral.github.io/qa/ (latest `main` of the source repo)

## How it works

The application/source code lives in the separate
[`cardscentral/cardscentral`](https://github.com/cardscentral/cardscentral) repo.
This repo only **builds and hosts** the site.

The workflow in [`.github/workflows/build-site.yml`](.github/workflows/build-site.yml):

1. Checks out this repo (the publish target) **and** the source repo.
2. Builds the PWA from source with the correct base paths:
   - QA from the source repo's `main` branch → `/qa/`
   - Prod from the source repo's latest release tag → `/app/`
   - Landing (static, from source `landing/`) → site root `/`
3. Commits the generated files into this repo's `main` branch, which GitHub
   Pages serves at the root domain.

Because it publishes to **itself**, it uses the built-in `GITHUB_TOKEN` — no
personal access token or deploy key is required.

## Triggers

- **Manual**: `Actions → Build & Publish Site → Run workflow` (pick which stages).
- **Scheduled**: refreshes QA (and picks up new releases for prod) automatically.
- **Source repo dispatch** (optional): the source repo can send a
  `repository_dispatch` to rebuild immediately on push/release.

## One-time setup

Repo **Settings → Pages → Source = "Deploy from a branch", Branch = `main` / `(root)`**.

## Do not edit generated files by hand

Everything except this README and the workflow is generated by CI. Manual edits
to `index.html`, `app/`, or `qa/` will be overwritten on the next build.
